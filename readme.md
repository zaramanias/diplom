# Labirint UI Automation Project

## Описание проекта

Проект представляет собой автоматизацию UI-тестирования сайта https://www.labirint.ru/ с использованием Selenium WebDriver.

Реализованы проверки:

- Поиск книги
- Добавление товара в корзину
- Удаление товара из корзины через кнопку "-"
- Проверка открытия карточки товара
- Очистка корзины
- Восстановление удалённых товаров

Проект построен с использованием Page Object Pattern.

---

## Используемые технологии

- Python 3
- Selenium WebDriver
- Pytest
- Allure
- WebDriverWait (явные ожидания)

---

## Архитектура проекта

Проект реализован по принципу разделения ответственности:

- `LabirintPage` — Page Object с локаторами и методами
- `test_ui.py` — тестовые сценарии
- Каждый тест содержит разметку шагов через Allure
- Используются явные ожидания (WebDriverWait + Expected Conditions)

---

## Стиль и форматирование

- Код соответствует стандарту PEP 8
- Используются docstrings
- Все действия размечены через @allure.step и with allure.step
- Применяется Page Object Pattern
- Используются явные ожидания вместо time.sleep

---

## Установка зависимостей

Выполните:

pip install -r requirements.txt

---

## Требования

- Установленный Google Chrome
- Установленный ChromeDriver (версия должна соответствовать версии браузера)
- Добавленный ChromeDriver в PATH

---

## Запуск тестов

Для запуска UI-тестов:

bash
pytest -m ui

Для запуска всех тестов:

bash
pytest

Для запуска с генерацией Allure-результатов:

bash
pytest -m ui --alluredir=./allure-results

После выполнения будет создана папка allure-results.

---

## Просмотр отчета Allure

Чтобы открыть отчет:

allure serve ./allure-results

Откроется браузер с детальным отчетом выполнения тестов.

---

## Что отображается в отчете Allure

- Название теста
- Описание теста
- Severity
- Пошаговое выполнение
- Статус выполнения (Passed / Failed)

---

## Реализованные сценарии

1. Поиск книги и проверка заголовка выдачи
2. Добавление нескольких товаров в корзину
3. Удаление товара через кнопку "-"
4. Проверка корректного открытия карточки товара
5. Очистка корзины и восстановление товаров

# Yougile API Automation Project

## Описание проекта

Проект представляет собой автоматизацию API-тестирования сервиса **Yougile** с использованием Python.

Реализованы проверки:

- Авторизация пользователя
- Создание проекта
- Создание доски
- Создание колонки
- Создание задачи
- Получение списка задач
- Получение задачи по ID
- Удаление задачи
- Удаление проекта

Проект построен с разделением логики API и тестов.

---

## Используемые технологии

- **Python 3**
- **Requests** — отправка HTTP-запросов
- **Pytest** — тестовый фреймворк
- **Allure** — генерация отчетов
- **python-dotenv** — работа с переменными окружения

---

## Архитектура проекта

Проект реализован по принципу разделения ответственности:

- `YougileApi` — класс для работы с API (Page Object для API)
- `test_api.py` — тестовые сценарии
- `pytest fixtures` — авторизация перед тестами
- `Allure steps` — подробная разметка шагов

---

## Стиль и форматирование

- Код соответствует стандарту **PEP 8**
- Используются **type hints**
- Методы документированы через `docstrings`
- Все действия размечены через `@allure.step` или `with allure.step`

---

## Переменные окружения

Перед запуском тестов необходимо создать файл `.env` в корне проекта и указать данные для авторизации:

LOGIN=your_login  
PASSWORD=your_password  
COMPANY_ID=your_company_id  
USER_ID=your_user_id  

---

## Установка зависимостей

Выполните:

bash
pip install -r requirements.txt  

---

## Запуск тестов

Запуск только API тестов:

bash
pytest -m api

Для запуска всех тестов:

bash
pytest

Для запуска тестов с генерацией результатов Allure выполните:

bash
pytest --alluredir=./allure-results  

После выполнения тестов будет создана директория `allure-results` с результатами выполнения.

---

## Просмотр отчета Allure

Чтобы открыть сформированный отчет, выполните:

bash
allure serve ./allure-results  

Команда запустит локальный сервер и автоматически откроет отчет в браузере.

## Что отображается в отчете Allure

- Название теста
- Описание теста
- Severity
- Пошаговое выполнение
- Статус выполнения (Passed / Failed)
